Primitive data types are the basic building blocks of data in a program. The primitive data types do represent
single values that are not objects. These types are directly stored in the stack memory, giving the user faster
access and manipulation.
On reference data types, they are also used to refer the objects holding multiple values and complex data
structures, which essentially points to the memory location where the data is stored. These types are stored in
the heap memory, where the variable holds an address of the actual data instead of the data itself.

Variable refers to the region of the program where the variable is accessible and can be used. Local variables
are usually defined within a function or a block of code and their scope is limited to that function or block, and
they can’t be accessed outside of it.
Global Variables are defined outside of any function / block and are accessible from anywhere throughout the
code, and this includes inside the functions.
Qn3
Memory Management: Some programming languages may reserve memory for a variable only when it is
initialized. This helps with efficient memory usage and management.
Control Flow: In conditional statements / loops, initialized variables can provide clear starting points for
calculations and logic. This helps to ensure flow of the program behaves as it should.
Preventing Errors: Using uninitialized variables can lead to runtime errors, unexpected results, or crashes.
Initialization ensures that a variable starts with a known state.
Defined Value: Initializing a variable gives it a specific value at the time of declaration. This helps avoid
undefined behavior that can occur when a variable is used without being set.
Clarity and Maintainability: Initializing variables improves code readability and maintainability. It allows other
developers (or your future self) to understand the intended use of the variable right away.

Static Variables: These are class-level variables, shared across instances, retaining their value throughout the
program.
Instance Variables: These ones are tied to individual instances, and they can have different values per object.
Local Variables: Temporary variables, which are scoped to a function or block. They are destroyed after
execution.
Qn5
Widening casting is the conversion of a smaller primitive type to a larger primitive type. The Java compiler
does this automatically. This is always safe with no data loss.
Narrowing casting is the conversion of a larger primitive type to a smaller primitive type. This requires explicit
conversion since there is a potential for data loss.

In Object-Oriented Programming (OOP), a class is a template used to create objects. It defines the properties,
attributes and behaviors that the objects created from the class will have.
Qn8
Importance of a Class in Java includes
 Encapsulation: Classes promote encapsulation, which is the bundling of data (attributes) and methods
that operate on that data into a single unit.
 Attributes: These variables hold the state of an object.
 Polymorphism: Classes can be designed to allow for polymorphism, where a method can operate
differently based on the object that it is acting upon.
 Methods: These are the functions defined within a class that describes the behaviors or actions that an
object can perform.
 Constructor: This is a special method that is called when an object of the class is created. It initializes
the object's attributes.
 Inheritance: Classes can inherit properties and methods from other classes, allowing for code reuse and
the creation of hierarchical relationships.
